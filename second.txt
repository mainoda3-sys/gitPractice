Main 브랜치 보호 설정:
Branch name pattern: main
Restrict pushes that create files: ☑️ 체크
Require pull request reviews before merging: ☑️ 체크
Require status checks to pass: ☑️ 체크 (선택사항)
Restrict who can push to matching branches: ☑️ 체크
개별 작업 브랜치 전략:
워크플로우:
개인 브랜치에서 자유롭게 작업
Pull Request로 main에 병합
코드 리뷰 후 승인되면 병합
이렇게 설정하면 main 브랜치는 안전하게 보호되고, 각자의 브랜치에서는 자유롭게 실험하고 연습할 수 있습니다!

브랜치 생성 명령어들:
1. 기본 브랜치 생성
  # 새 브랜치 생성
  git branch 브랜치명
  # 새 브랜치로 이동
  git checkout 브랜치명
2. 생성과 동시에 이동 (권장)
  # 새 브랜치 생성하고 바로 이동
  git checkout -b 브랜치명
3. 최신 Git 명령어 (Git 2.23+)
  # 새 브랜치 생성하고 바로 이동
  git switch -c 브랜치명
